## Ответы на контрольные вопросы

1. Чем отличается `var` от `let` и `const`?

`var` старее и имеет функциональную область видимости. Из-за этого переменная может быть доступна не там, где я ожидал. Еще `var` можно объявить повторно.

`let` имеет блочную область видимости. Это удобнее и безопаснее для обычных переменных.

`const` тоже имеет блочную область видимости, но значение нельзя переназначить после объявления. Если это объект или массив, то сам объект можно менять, но ссылку на другой объект уже нельзя присвоить.

2. Что такое неявное преобразование типов в JavaScript?

Это ситуация, когда JavaScript сам переводит значение из одного типа в другой. Например:

```js
"5" + 2 // "52"
"5" - 2 // 3
```

В первом случае число стало строкой, во втором строка стала числом.

3. Как работает оператор `==` в сравнении с `===`?

`==` сравнивает значения с преобразованием типов. Например, `5 == "5"` вернет `true`.

`===` сравнивает и значение, и тип. Поэтому `5 === "5"` вернет `false`. В обычном коде лучше использовать `===`, потому что так меньше неожиданных ошибок.
