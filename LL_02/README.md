Во второй лабораторной я написал свои версии методов для работы с массивами: `forEach`, `map`, `filter`, `find`, `some`, `every` и `reduce`.

Главное условие было не использовать готовые методы массива с такими же названиями. Поэтому вся логика сделана через обычный цикл `for`, обращение к элементам по индексу и `push`, когда нужно собрать новый массив.

## Что было сделано

- `printArray` и `printArray1` выводят массив в двух разных форматах;
- `forEach` вызывает переданный callback для каждого элемента;
- `map` возвращает новый массив с результатами callback;
- `filter` возвращает только элементы, которые прошли проверку;
- `find` ищет первый подходящий элемент;
- `some` проверяет, есть ли хотя бы один подходящий элемент;
- `every` проверяет, подходят ли все элементы;
- `reduce` накапливает один итоговый результат.

## Пример использования

```js
console.log(map([1, 2, 3], el => el * el));
// [1, 4, 9]

console.log(filter([1, 2, 3, 4, 5], el => el % 2 === 0));
// [2, 4]

console.log(reduce([1, 2, 3, 4], (acc, el) => acc + el, 0));
// 10
```

## Краткая документация

Все функции находятся в файле `script.js`. Каждая функция принимает массив и, если нужно, callback. Для функций, которые возвращают массив, создается отдельный `result`, чтобы не менять исходный массив.

Для `reduce` отдельно обработан случай, когда массив пустой и начальное значение не передано. В таком случае функция возвращает `undefined`.

## Ответы на контрольные вопросы

1. В чем преимущества использования колбэков при работе с массивами?

Колбэк позволяет отделить сам проход по массиву от действия, которое нужно выполнить с каждым элементом. Например, один и тот же `forEach` можно использовать для вывода, подсчета или проверки данных. Меняется только функция, которую мы передаем внутрь.

2. Какие проблемы могут возникать при использовании колбэков и как их избежать?

Главная проблема в том, что код может стать трудным для чтения, если колбэков слишком много или они сильно вложены. В асинхронном коде это обычно решают через `Promise` и `async / await`.

Еще одна частая проблема связана с `this`. Если передать метод объекта как обычный callback, контекст может потеряться. Это можно исправить через стрелочную функцию или `.bind()`.

3. Как реализовать `map`, `filter`, `find`, `some`, `every` и `reduce` без встроенных методов?

Нужно пройтись по массиву циклом `for`. На каждой итерации вызывается callback:

```js
callback(array[i], i, array);
```

Для `map` и `filter` создается новый массив. Для `find`, `some` и `every` можно остановить цикл раньше через `return`. Для `reduce` используется переменная-аккумулятор.
