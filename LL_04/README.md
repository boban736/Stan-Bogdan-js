В четвертой лабораторной я сделал небольшую модель инвентаря. В ней есть обычный предмет `Item` и оружие `Weapon`, которое наследуется от `Item`.

## Пример использования

```js
const bow = new Weapon("Longbow", 2.0, "uncommon", 15, 100);

console.log(bow.getInfo());
bow.use();
console.log(bow.durability);
bow.repair();
```

После вызова `use()` прочность оружия уменьшается. После `repair()` она снова становится `100`.

## Краткая документация

`Item` отвечает за общие свойства предмета. `Weapon` расширяет его и добавляет урон с прочностью.

Во второй части файла есть `ItemConstructor` и `WeaponConstructor`. Они делают почти то же самое, но через старый подход с прототипами:

```js
WeaponConstructor.prototype = Object.create(ItemConstructor.prototype);
```

## Ответы на контрольные вопросы

1. Какое значение имеет `this` в методах класса?

`this` указывает на конкретный объект, который вызвал метод. Например, если вызвать `bow.use()`, то внутри метода `use` значение `this` будет ссылаться на объект `bow`.

Если метод передать отдельно как callback, `this` может потеряться. Поэтому с методами классов надо быть внимательным.

2. Как работает модификатор доступа `#` в JavaScript?

`#` используется для приватных полей и методов. Такое поле можно читать и менять только внутри класса.

Например:

```js
class Example {
  #value = 10;
}
```

Снаружи обратиться к `#value` нельзя. Браузер выдаст ошибку.

3. В чем разница между классами и функциями-конструкторами?

`class` выглядит понятнее и короче. В нем проще писать наследование через `extends` и `super`.

Функции-конструкторы делают похожую работу, но там нужно вручную настраивать прототипы. Например, использовать `Object.create(...)` и восстанавливать `constructor`.

Еще класс нельзя вызвать без `new`, а функцию-конструктор случайно можно. Это может привести к ошибкам.
